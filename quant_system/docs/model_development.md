# 模型开发报告

## 1. 模型选择依据

### 1.1 LightGBM（基线模型）

**选择理由**：
- 天然处理非线性关系，自动捕捉特征交叉
- 训练速度快（比XGBoost快3-10倍），内存占用低
- 对缺失值鲁棒，不需要预填充
- 特征重要性输出，可解释性强

**适用场景**: 表格型因子数据，特征维度50-200

### 1.2 TabNet（进阶模型）

**选择理由**：
- 自适应特征选择（attention mask），自动筛选有效因子
- 每个决策步骤的可解释性（哪些特征被选中）
- 比全连接网络更适合表格数据

**适用场景**: 高维因子数据，需要自动特征选择

### 1.3 Transformer（时序模型）

**选择理由**：
- 捕捉因子值的时序动态模式
- 多头注意力机制可以发现不同因子之间的交互关系
- 位置编码捕捉市场状态的演变

**适用场景**: 有足够长的因子历史序列（至少60天）

### 1.4 Stacking集成

**选择理由**：
- 结合树模型（非线性）和深度学习（表示学习）的优势
- OOF预测避免了过拟合
- 元模型可以学习每个基模型的权重

## 3. 训练机制

### 3.1 滚动窗口 + 扩展窗口验证

```
[Train 5年] [Valid 1年]
    [Train 5年] [Valid 1年]
        [Train 5年] [Valid 1年]
```

每个窗口独立训练和验证，最终模型使用所有窗口的加权平均预测。

### 3.2 样本权重

$$w_i = w_i^{time} \times w_i^{vol}$$

时间衰减权重：
$$w_i^{time} = \alpha^{\frac{days\_from\_latest}{total\_days}}, \quad \alpha = 0.95$$

波动率倒数权重：
$$w_i^{vol} = \frac{1}{\sigma_{ret,i} + \epsilon}$$

### 3.3 损失函数

**IC加权MSE**:
$$\mathcal{L} = \frac{1}{N}\sum_i |IC_i| \cdot (y_i - \hat{y}_i)^2$$

其中 $IC_i$ 是样本i所在截面的因子IC。

**LambdaRank**:
$$\mathcal{L} = -\sum_{i,j: y_i > y_j} |\Delta NDCG_{ij}| \cdot \log(\sigma(\hat{y}_i - \hat{y}_j))$$

## 4. 特征重要性分析

### LightGBM特征重要性（预期排序）

1. momentum_60d / weighted_momentum（动量类）
2. hv_20d / downside_volatility（波动类）
3. ep_ttm / bp（价值类）
4. roe_change / roe_stability（质量类）
5. abnormal_turnover / north_flow_change（情绪类）

### TabNet注意力掩码

TabNet的mask机制可以提供每个决策步骤选择哪些特征。通过分析mask的均值和方差，可以判断：

- 高均值mask：始终重要的核心特征
- 低均值高方差mask：条件性重要的特征（某些市场状态才有效）

## 5. 防止过拟合

1. **时间序列交叉验证**: 不使用随机k-fold，而是按时间顺序划分
2. **样本权重**: 近期的样本权重更高，避免被历史数据主导
3. **早停**: LightGBM和TabNet/Transformer都使用验证集早停
4. **集成**: Stacking多个模型的平均预测天然降低方差
5. **特征数量控制**: TabNet的自适应特征选择去掉了冗余因子
6. **随机种子固定**: 确保结果可复现

## 6. 模型评估

| 指标 | 目标值 |
|---|---|
| 验证集IC | >0.05 |
| 验证集IR | >0.5 |
| 年化超额收益 | ≥8% |
| 最大回撤 | ≤20% |
| 信息比率 | ≥1.0 |

## 7. 模型更新策略

- **日频预测**: 每日收盘后运行模型预测，生成次日信号
- **月度重训**: 每月将新数据加入训练集，重新训练
- **季度因子复查**: 每季度评估因子有效性，剔除IC衰减的因子
- **半年度超参搜索**: 每半年进行一次完整的超参搜索
