# 系统架构设计

## 1. 技术选型

| 层次 | 技术 | 选型理由 |
|---|---|---|
| 数据获取 | akShare | 免费开源，覆盖A股全量数据，无需token |
| 数据存储 | Parquet + 日期分区 | 列式压缩，高效时间范围查询，无DB依赖 |
| 因子计算 | NumPy/Pandas 向量化 | GPU-free，内存高效，可复现 |
| 机器学习 | LightGBM | 处理非线性关系+特征交叉，训练快速 |
| 深度学习 | PyTorch (TabNet + Transformer) | 自适应特征选择 + 时序建模 |
| 回测引擎 | 自研事件驱动 | 适配A股T+1/涨跌停规则 |
| 配置管理 | YAML + 环境变量 | 简单可读，支持CI/CD |
| 测试 | pytest + pytest-cov | 覆盖率≥80% |

## 4. 关键设计决策

### 4.1 为什么自研回测引擎而不使用zipline？

- A股T+1制度：当日买入次日才能卖出
- 涨跌停限制：涨停无法买入，跌停无法卖出
- 印花税仅卖出单边征收
- 这些规则在zipline中需要大量hack

### 4.2 为什么使用Encoder-only Transformer？

- 收益率预测是一个回归问题，不需要seq2seq
- Encoder + pooling + linear head 已经足够
- 减少参数量和训练时间

### 4.3 因子注册表模式

```python
@register_factor("momentum_20d", "momentum", "20日收益率")
class Momentum20D(BaseFactor):
    def compute(self, ...) -> FactorResult:
        ...
```

优点：
- 新增因子只需要一个类和装饰器
- 无需手动维护因子列表
- 支持按分类批量计算

## 5. 部署架构

```
单机部署:
  Python 3.11+ + 32GB RAM + 500GB SSD

数据流:
  crontab定时任务 → akShare获取增量 → 清洗 → 入库
  模型重训练: 月度滚动训练

监控:
  - 数据缺失告警
  - 模型预测异常检测（预测值分布偏移）
  - 回测绩效跟踪
```
